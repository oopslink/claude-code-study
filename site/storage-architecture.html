Claude Code 存储架构完全指南

涵盖：设置系统 / CLAUDE.md / Skills / Transcript / History / Memory / Undo / Plans / 运行时 / 缓存 / Worktree

核心概念

所有持久化数据围绕一个根目录展开：

~/.claude/     # 默认根目录，可通过 CLAUDE_CONFIG_DIR 覆盖
▶ 三套同构的四层结构
整个体系由三套并行的四层结构组成：设置系统、CLAUDE.md 指令体系、Skills 体系。三套都遵循同一个优先级模型：
Managed
→
User
→
Project
→
Local
后加载 = 高优先级

右边优先级更高
Source文件路径备注
userSettings~/.claude/settings.json用户全局
projectSettings<repo>/.claude/settings.json可提交 git
localSettings<repo>/.claude/settings.local.jsongitignored 放 secrets
flagSettings--settings 启动参数内存，不落盘

policySettings 自身四源合并

1. 远程 managed settings（API 下发）               ← 最高
2. 平台 MDM（macOS plist / Windows HKLM）
3. managed-settings.json + managed-settings.d/*.json  ← 按文件名排序
4. HKCU（Windows 用户注册表）                       ← 最低

Managed 根目录（平台相关）

macOS:   /Library/Application Support/ClaudeCode/
Linux:   /etc/claude-code/
Windows: C:\Program Files\ClaudeCode\
▶ localSettings 与 worktree
settings.local.json 是 gitignored 的。worktree 创建时会 copy（不是 symlink）到 worktree 的 .claude/ 目录——symlink 会共享状态，copy 才能做真正的隔离。

三、指令文件系统（CLAUDE.md 体系）

核心文件：src/utils/claudemd.ts　·　越晚加载优先级越高

# 加载顺序（从低到高优先级）
1. /etc/claude-code/CLAUDE.md                          Managed
2. /etc/claude-code/.claude/rules/*.md                 Managed rules
3. ~/.claude/CLAUDE.md                                 User
4. ~/.claude/rules/*.md                                User rules
5. <每级目录>/CLAUDE.md                               Project，root → cwd
6. <每级目录>/.claude/CLAUDE.md                       Project
7. <每级目录>/.claude/rules/*.md                      Project rules
8. <每级目录>/CLAUDE.local.md                         Local，gitignored
9. ~/.claude/projects/<cwd>/memory/MEMORY.md          memdir（若启用）
10. ~/.claude/projects/<cwd>/memory/team/MEMORY.md    团队记忆（若启用）

@include 指令

# CLAUDE.md 内可引入其他文件
@./relative/path
@~/home/path
@/absolute/path

循环引用被检测并跳过；项目目录外的引入需用户明确授权。

嵌套 Worktree 去重

在 .claude/worktrees/<slug>/ 内运行时，向上遍历经过 worktree 根和主仓库根。Project 类型文件只从 worktree 自身 checkout 读取，避免重复加载。

六、输入历史

核心文件：src/history.ts、src/utils/pasteStore.ts

~/.claude/history.jsonl          # 全局输入历史（追加写入，lockfile 保护）
~/.claude/paste-cache/<hash>.txt # 大文本粘贴，SHA256[:16] 命名

两级存储策略

粘贴大小存储方式
≤ 1024 字节内联在 history.jsonl 条目内
> 1024 字节存入 paste-cache/，history.jsonl 只存 hash
图片单独走 image-cache/ 路径，不进 history.jsonl

写入机制

追加写入（appendFile）→ lockfile 串行化 → 内存缓冲 pendingEntries
→ 异步 flush → cleanup hook 兜底（进程退出时确保落盘）
Esc 中断：从 pendingEntries 撤销；若已 flush 则加入 skippedTimestamps 跳过

七、Memory 系统（memdir）

核心文件：src/memdir/memdir.ts

~/.claude/projects/<cwd>/memory/
├── MEMORY.md          # 索引（上限 200 行 / 25KB）
├── user_role.md       # topic 文件
├── feedback_*.md
└── team/              # Team memory（TEAMMEM 功能）
    └── MEMORY.md

四类型分类法

类型用途
user用户角色、偏好、知识背景
feedback对 Claude 行为的修正与肯定（含 Why / How to apply）
project项目背景、目标、决策（含 Why / How to apply）
reference外部系统指针（Linear、Grafana 等）
▶ MEMORY.md 约束
是索引不是内容。超过 200 行或 25KB 时截断并附 warning，提示压缩。各 topic 文件带 YAML frontmatter（name / description / type）。

KAIROS 模式（长期 assistant session）改为 append-only 日志：logs/YYYY/MM/YYYY-MM-DD.md，由 nightly /dream 蒸馏成 MEMORY.md。

八、文件编辑历史（Undo）

核心文件：src/utils/fileHistory.ts

~/.claude/file-history/<session-id>/<fileNameHash>@v<version>

每次 Claude 编辑文件之前，先把当前内容备份到此目录。文件名：sha256(filePath)[:16] + "@v" + version。

--resume 时，旧 session 的所有备份会 hardlink 迁移到新 session 目录，确保 undo 跨 session 有效。

九、Plans（计划模式）

核心文件：src/utils/plans.ts

~/.claude/plans/<slug>.md               # 主对话计划
~/.claude/plans/<slug>-agent-<id>.md    # 子 agent 计划

如果项目 settings.json 配置了 plansDir，则存到项目内路径。/clear 时重置 slug 生成新文件。

十、运行时状态

并发会话注册

~/.claude/sessions/<pid>.json   # claude ps 的数据源

每个进程写 PID 文件，记录 sessionId、kind（interactive/bg/daemon）、status（busy/idle/waiting）。崩溃后下次启动扫描清理僵尸文件。

图片 / 上传 / Shell 快照

~/.claude/image-cache/<session>/   # 粘贴图片，按 session 隔离
~/.claude/uploads/<session>/       # IDE bridge 上传的附件
~/.claude/shell-snapshots/          # Bash 工具跨调用保持 shell 状态

十二、安全存储（凭证）

核心文件：src/utils/secureStorage/index.ts

macOS:   Keychain → 失败时降级到 plainTextStorage
Linux:   plainTextStorage （TODO: libsecret）
Windows: plainTextStorage

Fallback 策略：macOS 优先写 Keychain，失败自动降级，不阻断启动流程。

十三、Worktree 存储

核心文件：src/utils/worktree.ts

<repo>/.claude/worktrees/<slug>/      # git worktree 实体
└── .claude/
    └── settings.local.json           # 从主仓库 copy

嵌套 slug（user/feature）展平为 user+feature，避免 git ref D/F 冲突与父目录删除联带删子问题。

设计规律总结

三套同构四层
设置、CLAUDE.md、Skills 都是
Managed → User → Project → Local
追加写入
history.jsonl、session .jsonl
避免随机写，支持多进程并发
内容寻址
paste-cache/ 用 SHA256 命名
天然去重，覆写安全
lockfile 保护
history.jsonl 写入时加锁
防多进程交叉写入
两级粘贴存储
小内联大外存
history 文件不因大粘贴膨胀
session 隔离
image-cache、file-history、uploads
都按 session 子目录隔离
copy 不 symlink
secrets（settings.local.json）必须 copy
大无状态目录（node_modules）才 symlink
XDG 规范
cache 和 config 分离
平台路径由 env-paths 统一管理
